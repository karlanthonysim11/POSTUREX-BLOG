How It Works

📍

1. Pressure Capture

Four FSR 402 sensors detect seat pressure patterns in real time.

📏

2. Distance Check

The HC-SR04 sensor measures user distance and sitting presence.

🧠

3. Local Processing

ESP32 (NodeMCU/DevKit) analyzes posture data using rule-based logic.

📳

4. Haptic Alert

A disc vibration motor gives instant correction feedback when posture is poor.

📶

5. Wireless Sync

ESP32 sends posture status to the mobile app over Bluetooth.

📱

6. Dashboard Insight

The app displays live posture state, alerts, and weekly trend summaries.

Technology Stack

⚙️

ESP32 Microcontroller

NodeMCU/DevKit for sensor reading, logic, and Bluetooth data transfer.

📍

4x FSR 402 Sensors

Pressure sensing layer to detect sitting balance and posture shifts.

📏

HC-SR04 Ultrasonic Sensor

Distance sensing for user presence and seated-position validation.

📳

Haptic Driver Circuit

Disc vibration motor with 2N2222 transistor and 1N4001 diode protection.

🔋

Power System

5V rechargeable power bank or 9V battery setup for portable use.

📱

Android Mobile App

Real-time posture dashboard, alerts, and weekly analytics view.

AI Technology

Lightweight Rule-Based AI Engine

Adaptive Thresholding: The system adjusts sensitivity based on user pressure patterns and sitting behavior.

Real-time Analysis: ESP32 processes sensor input continuously to detect posture changes immediately.

Three Posture Classes: Upright (good), Slouched (warning), and Leaning (alert) for actionable guidance.

Built for edge computing: fast response, low cost, and no cloud dependency.
